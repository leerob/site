## Images

I gave a short talk about the Next.js Image Component at [Jamstack Toronto](https://twitter.com/MHillestad/status/1351931891176247296) with some fine folks from Gatsby, Netlify, and Sanity. No recording yet, but I've compiled some best practices for image performance on [Twitter](https://twitter.com/leeerob/status/1352264153411497993). In summary:

1. [Use `width` and `height` to prevent layout shift](https://twitter.com/leeerob/status/1352264154481045506)
2. [Lazy-load images as they enter the viewport](https://twitter.com/leeerob/status/1352264155902963712)
3. [Use modern image formats (WebP, AVIF)](https://twitter.com/leeerob/status/1352264157228388352)
4. [Serve correctly sized images using `srcset`](https://twitter.com/leeerob/status/1352264158281129986)
5. [Provide blur-up placeholders](https://twitter.com/leeerob/status/1352264162248957953)

Check out the [full thread](https://twitter.com/leeerob/status/1352264153411497993) for more details (along with [my video](https://www.youtube.com/watch?v=h0gj4gOjz44&feature=youtu.be) if you missed it).

### Web Fonts

It's easy to forget typography accounts for [95% of web design](https://ia.net/topics/the-web-is-all-about-typography-period/). Performant font loading is critical for improving your Core Web Vitals. Here's how to get high-performance web fonts in 2021:

1. [Use a variable font](https://twitter.com/leeerob/status/1345554814248374273)
2. [Preload your font file](https://twitter.com/leeerob/status/1345554815158546432)
3. [Self-host instead of Google Fonts](https://twitter.com/leeerob/status/1345554816140005379)
4. [Use `font-display: optional` to prevent layout shift](https://twitter.com/leeerob/status/1345554817092083712)

Check out my video [Web Fonts in 2021](https://www.youtube.com/watch?v=G0cOQ79WKZE) for more details. It's one of my best yet, in my opinion. It's also [available as a blog post](https://leerob.io/blog/fonts) if you prefer.
